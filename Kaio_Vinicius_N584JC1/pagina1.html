OQUE É UM DER

No contexto de banco de dados, DER é uma sigla que significa "Diagrama de Entidade-Relacionamento". O Diagrama de Entidade-Relacionamento é uma ferramenta de modelagem de dados amplamente utilizada na engenharia de software e em projetos de banco de dados. Ele é usado para representar a estrutura de dados de um sistema e os relacionamentos entre as entidades (tabelas) desse sistema.

Os principais elementos de um DER incluem:

  1. ENTIDADES: Representam objetos ou conceitos do mundo real que precisam ser armazenados no banco de dados. Cada entidade é representada por uma tabela no banco de dados.

  2. ATRIBUTOS:São características ou propriedades das entidades. Os atributos são representados nas colunas das tabelas.

  3. RELACIONAMENTOS: Descrevem como as entidades estão conectadas ou associadas umas às outras. Os relacionamentos são representados por linhas ou setas que ligam as entidades no diagrama.

  4. CAHVES PRIMARIAS E CHAVES ESTRANGEIRAS São utilizadas para estabelecer as relações entre as entidades. Uma chave primária é um atributo que identifica exclusivamente uma linha em uma tabela. Uma chave estrangeira é um atributo em uma tabela que faz referência à chave primária de outra tabela, estabelecendo assim uma ligação entre as duas tabelas.

  5. CARDINALIDADE Indica quantos registros em uma entidade estão relacionados a quantos registros na outra entidade. Pode ser "um para um" (1:1), "um para muitos" (1:N), "muitos para muitos" (N:N) e assim por diante.
